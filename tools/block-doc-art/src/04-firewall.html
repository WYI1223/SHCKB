Type is the guardrail · 实现

类型即护栏 — 写不出的属性，就破坏不了布局

skin 的 box.style 不是任意 CSSProperties，是一个只含视觉属性的 Pick<> 子集。结构属性在类型里根本不存在 → 编译期就拦住，不靠约定、不靠 review。

SkinBoxStyle = Pick<CSSProperties, …> — skin / 第三方插件能写的全部
✓ 可表达（纯视觉，skin 自由发挥）
backgroundbackgroundImagebackgroundSize borderborderLeftborderRadiusborderImage paddingcolorboxShadow fontSizelineHeightscrollbarWidth
✗ 类型里没有（写出来通不过编译）
positioninsettopleft widthheightoverflow displayflexzIndextransform
→ 结构归 host：position · width · height · overflow 由 BlockFrameCore 最后落 （skin 视觉先铺，host 结构后盖，skin 赢不了）

不可表达 = 不可破坏。 插件作者物理上写不出破坏布局的属性 → stationery 那种「绝对定位塌成 0 高」的塌缩，构造上不可能再发生——对第三方 skin 也一样。逐 theme × kind × skin 的不变量测试只是兜底；真正的护栏是类型本身。
