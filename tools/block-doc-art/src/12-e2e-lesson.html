Why · the frame must stay transparent to link clicks

单测全绿，真浏览器才抓到

委托处理器挂在外层滚动容器上。但编辑器里 block 帧的激活 onClick 无条件 stopPropagation()——把链接点击在冒泡途中吞了，于是裸 /p/:id 又整页重载（F4 的 bug 复活）。

前 · 吞掉点击 后 · 让路
✗ 前 — 帧吞掉了点击
点链接→ 冒泡到 .pu-block onClick→ stopPropagation() ✋→ 委托处理器收不到→ 默认导航 /p/:id → 302
✓ 后 — 对内部链接透明（设计约束）
onClick={(e) => {
  if (!isActive && e.target.closest('a[data-skb-page]')) return;  // 让路
  e.stopPropagation();                // 选区/激活照常
  if (!isActive) onActivate(block.id);
}}
