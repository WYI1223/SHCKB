Root cause · layer-error

根因 = 把「永链」当成了「导航」

/p/:id 是公开分享永链（改名不烂、不漏私有）。它被误用成编辑器正文里的导航——两层塌成一层，就是 bug。

前 · layer-error 后 · 保面导航
✗ 前 — 永链当导航
点击→ 裸 <a href="/p/:id">→ 服务器路由（不在 SPA router）→ 302→ published /notes/:slug
整页重载
草稿 / 私有目标 → 404
永远落发布快照（旧）
✓ 后 — 同一个 LinkRef 走导航
点击→ 委托处理器→ navigateToPage(LinkRef)→ resolveTarget(当前面)→ 客户端跳转 · 镜头守恒
🧭
分层：永链 = 身份（分享 / 引用 / 改名稳定） · 导航 = 移动（应用内、保面）。把两层塌成一层，就是这个 bug 的根。
