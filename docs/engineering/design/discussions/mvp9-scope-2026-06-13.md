3. **页间一等链接搭车**：链接 mark 存 pageId（改名不断链），随 richtext 落地。

## 设计桩

**插件姿势纪律（压力测试的测法）**：richtext 整个住
`packages/block-kinds/src/richtext/`，只许消费 plugin 能拿到的面——
registry 注册、HostServices、ThemeContext、ui-kit。**禁止** import apps/web
的任何东西（chrome/overlays 不可达——这是故意的：插件够不着 chrome）。
凡是它需要而契约没有的能力，就是 plugin system 的一个 finding，
加进 HostServices（可选字段，旧 host 不破）并记录。

**服务端导入安全**：publish 管线（static.ts → registry → 各 kind）在 Bun
无 DOM 环境跑。RenderView = 纯 PM-JSON → React 树行走器，**零 prosemirror
依赖**；prosemirror-view 只在 EditView 的 effect 里动态 import。

**内容格式**：`{ doc: <ProseMirror JSON> }`，自定义小 schema（与渲染行走器
锁步）：paragraph / heading 1-3 / bullet·ordered list / blockquote /
hard_break + marks strong / em / code / link(href) / **pagelink(pageId)**。

**页间一等链接**：pagelink mark 只存 pageId（身份），不存 slug（地址）；
渲染为 `/p/:pageId` 永链；服务端新路由 `GET /p/:id` → 已发布公开页 302 到
当前 `/notes/:slug`，否则 404（与 slug 路由同款 no-leak）。
当前 slug 政策下（首发锁定、永不变 [publish 端点注释]）"一等"的实际收益：
① 可以链接**尚未首发**的页面——首发前 slug 还是建页占位，存 slug 必断，
存 pageId 则目标首发后 302 自动接通；② 对未来任何 slug 政策变化免疫。
显示文本不自动跟改名（backlink/自动标题 = 后续轮）。
