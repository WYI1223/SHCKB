## 决策摘录

- **M7-D1 顺序**：① review（修高优）→ ② 产品内文档（与③可并行）→ ③ UI 分叉 → 收束对比评审。文档写在 review 修复之后（描述修复后的现实）。
- **M7-D2 review 形态**：按维度并行派 review subagent（安全面 / 契约漂移 / 错误处理一致性 / 测试盲区 / 依赖与构建健康），产出分级 findings → owner 裁决修哪些。
- **M7-D3 docs-as-bundle**：开发者导览/架构说明/QA 写在产品 dev 库，**确定性导出存回 git**（examples/ 同款模式）——canonical 真理仍在 git（PRD-master 纪律），产品内是工作副本兼展示；PRD/ADR 治理文档不搬。预期同时收获 dogfooding 摩擦清单（页间链接/搜索/长文/code 编辑器痛感）。
- **M7-D4 UI 分叉研究**：3 个 subagent 各在隔离 worktree 出完整 UI 方向，**brief 不含现 UI 截图**（无偏好的实操保证），产出可跑分支+截图集+设计说明；owner 对比择采。隐藏目标：为 chrome 放权积累形态样本——chrome 槽位契约在见过 N 种形态后再冻结（"3 themes = reference impl"同款教训）。
- **M7-D5 远期注记（零成本）**：plugin-system PRD 注"画布列数/可堆叠 block = engine 级扩展点候选（难点 = 覆写 engine）"；theme-system PRD 注 chrome/骨架放权哲学。
- **M7-D6 review 修复裁决**（owner 2026-06-12）：findings **全修**（🔴🟡🟢 全量，见 mvp7-review-findings）；②③ **串行**（文档先、UI 分叉后）；UI 分叉三方向之一改为**完全不给方向**——只给产品形态，放开做整体优化，其余两个保留方向调性。
