Owner 提出四项诉求（2026-06-12 对话）：0. 场景案例（日记 / 旅行照片 / 知识点+代码解析）；1. 通用工具面板（任何插件可使用）；2. 字体/颜色更改；3. 更高自定义化主题（表面材质颜色 / 主题配对调色板 / 更深层自定义）。Owner 判断通用面板是 plugin-system 主线前的障碍之一——成立。

## 决策摘录（详见 spec）

- **M5-D1 场景案例先行 = 需求探测器**：三案例（日记→stationery、旅行照片→拍立得/image、代码解析→workbench/code）把现有 kind 和主题推到真实负载下，暴露的摩擦构成 1/3 号的真实需求清单。建在 owner 当前 dev 库（owner 拍板），独立文件夹，产品内容非测试垃圾。
- **M5-D2 案例导出进 `examples/`**：用 MVP-3 确定性导出存 v2 包——同时是产品 demo、主题测试夹具、新实例一键导入示例包。git 友好格式的第一个仓库内活体。
- **M5-D3 自定义 = 主题授权的数据，不是绕过主题的后门**（owner 2/3 号合并）：自由调色（3a 无约束版）**rejected**——破坏主题视觉完整性 + QA 面爆炸。模型：主题声明 `palettes?: PaletteVariant[]`（科研 colormap 式官方配色变体）+ token 覆写 **opt-in 白名单**（主题声明哪些 token 开放）。与 [theme-extensibility：theme = plugin type、主题作者有主权] 一致。
- **M5-D4 工具面板分权规则**：模块声明工具（`BlockKindModule.tools`），host 决定面板位置与形态——主题轮"几何/视觉分权"哲学在 chrome 上的复用。**插件管面板内容，host 管面板位置和布局。**
- **M5-D5 提前实现的通用组件**（"≥2 个消费者"标准）：主题化表单原语（Select/Button/TextInput/Toggle——code/image kind 已各写一遍，调色板选择器是第三消费者）、工具面板贡献点、调色板选择器、HostServices 扩档（向工具组件传激活块+内容读写）。**不**提前做：Modal 体系、通用文件选择器、插件设置页（等 plugin-system 正轮）。
- **M5-D6 面板放权清单**：DRAG TO INSERT 正式化为纯 registry 驱动（kind 自带 label/glyph/分类，零硬编码）；工具面板内容全放权；块头部 chrome host 保留但读模块元数据；侧栏/主题选择/Export-Import host 保留（实例与页面级关切）。
- **M5-D7 顺序 0 → 1 → 2+3**：0 喂 1 和 3 的需求；1 的原语被 3 复用。自定义选择进 manifest.settings = 又一次 format 纪律实战（成对 transform + 降级行为）。
