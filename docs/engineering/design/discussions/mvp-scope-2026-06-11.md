## Ratified Decisions

### D1 — MVP definition

Local single-user SHCKB instance proving the vertical slice: **create notepage → edit (grid + markdown blocks) → publish → read**. This is below M2 (no auth, no onboarding gate) but every piece sits on the product-shaped architecture so nothing is disposable.

### D2 — Two-state model ships in MVP

Author working state vs last completed public state ([notepage.md] core decision) is implemented in the MVP data model: normalized working state + published snapshot + explicit update-public action. Rationale: it is the single most load-bearing PRD invariant and a schema-level decision; retrofitting later means data migration plus full read/write path rewrite, while MVP cost is low (one snapshot column + one publish endpoint).

### D3 — No login; PEP seam with stub principal

MVP has no identity machinery. A PEP middleware exists and injects a fixed local author principal as immutable `ctx.user` ([pep.md] seam). Visibility (private/public) and route classes ([notepage.md]) are implemented; "anonymous reader" is simulated by the public read route not requiring the principal.

### D4 — "打框架" means seams, not systems

For each non-core PRD area the MVP carries the **interface where the PRD boundary lives, with zero machinery behind it**:

| PRD area | MVP form |
|---|---|
| Blocks / plugin-system | `BlockKindModule` interface + markdown as the only implementation + static registry object. No runtime registry/lifecycle/sandbox. |
| Theme-system | graph-paper visuals only, passed through a theme-tokens object. No cascade, no switcher. |
| Responsive | Desktop only. Logical 12-col GridState is itself the seam. |
| Self-host-deploy | `compose.dev.yaml` run-verification path only. No install bootstrap. |
| Search / AI / discussion | No code. |

Rationale: AGENTS.md check 4 (no over-abstraction before real product need). Pre-building registries/cascades/middleware-chains "for later" is the failure mode to avoid.

### D5 — Stack baseline

Captured as [ADR-0019] (PRD-informed; explicitly **not** a revival of deprecated [ADR-0006] even where conclusions overlap): Bun runtime + workspaces, Hono server, Drizzle + SQLite, React + Vite, react-markdown/remark-gfm/rehype-sanitize at spike level, vitest.

### D6 — Environment: two layers (owner ratified 2026-06-11)

- **Dev inner loop**: native (Windows host for now; Bun 1.3.14 installed on host). Container-free for fast test/HMR cycles.
- **Run verification**: Docker compose path (`compose.dev.yaml`) exercising the operator-shaped runtime early.
- Dev-environment files (Dockerfile base image etc.) are dev convenience, **not** stack ratification; runtime base image choice belongs to later ADR work.

### D7 — Gravity toggle is an author-facing product function

Owner ratified (2026-06-11) that the prototype's gravity on/off switch was intended as an author choice. MVP persists `gravity_enabled` per notepage (page metadata, outside GridState). PRD pass to absorb this into [notepage.md]/[notepage-editing.md] is pending; grid-engine CONTRACT "always gravity-stable" wording must be revised at lift time (a gravity-off page may persist non-gravity-stable layouts). Theme L0a lock narrows to "gravity toggle belongs to author only; theme/plugin may not touch it."

### D8 — Grid-engine lifts to `packages/grid-engine` at target contract

Source lifts from `carryover/packages/grid-engine/src/` with the target contract applied: `BlockKind = string` (open identifier, kind-opaque), default-size lookup moved out of the engine to the block-kind module layer. All carryover tests must pass after the lift. CONTRACT.md cleanup (ProseMirror/editor-shell/ADR-0019-name-collision residue) happens in the same change.

### D9 — Decision recording protocol

In-flight implementation decisions during the MVP build are appended to this record (dated entries under "Build Log"); stack-level changes go to ADR-0019 or a new ADR.

## MVP Scope (included)

- Notepage list + create (default private, per [notepage.md])
- Authoring route: 12-col grid editor lifted from prototype interaction layer + graph-paper visuals; insert/move/resize/delete blocks; author gravity toggle
- Markdown block per [block-markdown.md] M2 shape: inactive = rendered preview, active = source-plus-preview lightweight editor, focus-leave hands content to notepage host; bounded mounted editor count
- Working state autosave (debounced whole-state PUT); server re-validates layout invariants via shared grid-engine `validateState`
- Explicit update-public action promoting a published snapshot; private/public toggle
- Public read route `/notes/:slug` rendering the published snapshot (route-class semantics; client-rendered, SSR deferred)

## Out of MVP (explicitly)

Login/identity, SSR/SEO, preview route, theme switching, responsive/touch, undo/redo, delete-tombstone semantics beyond simple delete, backup/health endpoints, plugin runtime machinery, search, AI.

## Open Items

- SSR for public read route: deferred; revisit before M2 (Lighthouse gate assumes SSR-quality read path).
- Slug strategy: MVP uses generated slug from title + uniqueness suffix; alias/tombstone rules remain future PRD work.
- Gravity-toggle PRD pass ([notepage.md] data boundary + re-enable collapse semantics) — pending owner-scheduled PRD round.
- Markdown package choices remain spike-level per [block-markdown.md]; replacement boundary must hold.
