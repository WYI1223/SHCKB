# MVP-6 design — Properties 轮

Status: owner ratified（2026-06-12 对话："没问题，了解了，这东西确实通常叫properties"）；
决策 M6-D1..D7 见 [mvp6-scope-2026-06-12.md](../../engineering/design/discussions/mvp6-scope-2026-06-12.md)。

## Goals

1. **Selection 模型 + Properties 面板**（停靠于侧栏目录下方，3D 引擎检查器式）：点页面背景看页面属性，点 block 看块属性
2. **作者级外观（第三自定义轴）**：theme 策展块壳选项（纸→白卡→无框）+ 页面背景（颜色/图片）
3. **还 ADR-0026 挂账**：theme 槽位组件渲染时读 token（解 TDZ 结构）→ stationery palettes 解锁
4. **Format v4** 成对 transform 承载新外观数据

## Non-goals

- 右键悬浮菜单（M6-D2 后做——同一属性区块的快捷投影，内核稳定后是糖）
- richtext / text-range selection（仅类型占位）
- 移动现有 chrome（页面钉选 select 留在编辑器头部，不搬进 Properties）
- 块壳自由样式（颜色/边框直改）——壳选项必须 theme 策展（M6-D3）

## 1. Theme 包结构重构（TDZ 解链，M6-D6 前提）

现状循环：`themes.ts(THEMES) → stationery.tsx → context.tsx → themes.ts(graphPaper)`。
拆 `registry.ts`：

- `themes.ts`：类型 + graphPaper/ink + helpers（**不 import 任何主题文件**）
- `context.tsx`：只 import themes.ts（graphPaper 缺省）
- `workbench/stationery/blueprint`：import 类型（erased）+ context（useTheme）——**槽位组件改为 useTheme() 渲染时读 token**，TOKENS 常量仅用于构造主题对象
- `registry.ts`：组装 `THEMES`/`DEFAULT_THEME_ID`（import 所有主题文件）；index.ts 重导出，消费者无感

效果：palettes 对深槽位主题真实生效（stationery 解锁变体——本轮顺带给 1 个：`kraft` 牛皮纸调）。

## 2. 块壳选项（theme 策展）

```ts
// themes.ts
export type ShellOption = {
  id: string;          // 进 block.shell 持久化，不改名
  name: string;
  /** 适用 kind 白名单；缺省 = 全部 kind */
  kinds?: string[];
};
Theme.shellOptions?: ShellOption[];
// BlockFrameProps 增：
shell?: string | null;   // 作者选择的壳选项 id；null/未知 id = 主题默认壳
```

- 策展首批：stationery `paper`(默认)/`card`(白卡片，无撕边无胶带)/`bare`(无底纸——image 即"只显示照片")；workbench `outlined`(默认)/`flat`(无边框融入画布)
- 未知 shell id → 默认壳（主题升级移除选项时页面照常渲染，与 paletteId 同纪律）
- DefaultBlockFrame 忽略 shell（token-only 主题零迁移）

## 3. 页面背景

```ts
type PageBackground = { color?: string; blobHash?: string };  // notepages.background JSON
// CanvasSurfaceProps 增：
background?: PageBackground | null;
```

- 缺省 surface / PublishedCanvas 根：color 替换 canvasBg；blobHash 以 cover 图层垫底
- 深槽位主题的 surface 自行诠释（主题主权：stationery 把图垫在桌面纹理下或替换之）
- blob 引用：background.blobHash 必须进 blob 引用枚举（export/GC 契约 [ADR-0023]）

## 4. 存储（migration 0007，schema v7）

- `blocks.shell TEXT NULL`；working-state PUT 的 block 对象增可选 `shell`（string|null，校验）
- `notepages.background TEXT NULL`（JSON）；新端点 `POST /api/notepages/:id/background`
  （body `{ color?, blobHash? } | null`；校验 blobHash 存在于 blobs 表；改后若已发布即时重渲染该页——与钉选同语义）
- PublishedDoc 增 `background?` + blocks 携带 `shell`（publish 快照即含外观，f(doc,…) 纯度不变）

## 5. Selection 模型 + Properties 面板（apps/web）

```ts
type Selection = { type: 'page' } | { type: 'block'; blockId: string };  // future: text-range
```

- EditorPage：`selection` state 取代裸 activeId（activeId = selection.block 的派生）；点画布空白 = page selection（原"取消激活"行为不变，只是显式化）
- **停靠位**：Sidebar 底部渲染稳定锚点 `<div data-skb-properties-slot>`；EditorPage 经 createPortal 注入 Properties 面板（侧栏折叠 = 无面板，可接受）
- 区块来源（M6-D1）：
  - page 选中 → host 区块（背景颜色 input + 背景图上传/清除，经 useHost().uploadBlob）
  - block 选中 → theme 区块（壳选项 chips，仅当当前有效主题为该 kind 策展了选项）+ module 区块（既有 tools）
- 现 `ToolPanel.tsx`（悬浮固定面板）退役，内容并入 Properties 面板

## 6. Format v4

- page.json 增 `background`（可空）；blocks[] 增 `shell`（可空）
- up(v3→v4)：补 null（无损）；down(v4→v3)：丢 background/shell，逐项报损失
- importer：shell 为字符串或 null 校验；background.blobHash 必须在 bundle blob 集合内

## 7. 测试

- theme：registry 重构后既有 10 测试不动；shellOptions 契约（未知 id 降级）；stationery kraft 变体经 applyCustomization 真实变 token
- block-kinds：Frame 收 shell prop 渲染分支（stationery card/bare）；静态渲染含背景层
- server：migration v7；background 端点（404/校验/重渲染）；publish 快照含 shell+background；format v4 round-trip + down 损失 + v3 包 up 导入
- Playwright：点画布 → 页面属性出现；改背景色画布即变；点 image 块 → 壳选项 bare → 底纸消失；发布页同步

## 风险

- theme 包重构动 import 图 → **vite 必须重启**（惯例坑）；Bun TDZ 用 `bun run` 实测验证（vitest 容忍度更高，不可作准）
- 双向数据：block.shell 走 working-state 全量 PUT（已有通道）；background 单独端点（页面级，与钉选同形）
- publishedHtml 纯度：背景/壳全部进 doc 快照，f(doc, slug, effectiveTheme) 签名不变
