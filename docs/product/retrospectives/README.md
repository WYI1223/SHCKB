## Retrospective 写作规则

- 一个 retrospective = 一个事件 / 一段时间 / 一类失败
- 必含：**发生了什么 / 为什么发生 / 学到什么 / 如何避免下次**
- 不修改既有 retro；新教训 → 新 retro doc + 更新本 README index
- Filename pattern：`<topic>-<YYYY-MM>.md` 或 `<event>-<YYYY-MM-DD>.md`

## 何时写 retro

- 决策失败 + 复盘清楚原因后
- Framing 错误被发现后（如本目录第一份 retro）
- 系统级 incident 后（性能 / 数据 / 安全）
- Wave / milestone close 时（如有教训积累）
